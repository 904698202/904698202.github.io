Webpack

问：什么是Webpack？

答：本质上来讲，webpack是一个现代的Javascript应用的静态模块打包工具，能在模块化开发时帮助我们处理模块间的依赖关系。

和grunt/gulp的对比

问：grunt/gulp是什么？

答：grunt/gulp的核心是Task，我们可以配置一系列的task，并且定义task要处理的事物，如ES6、ts转化，图片压缩，scss转成css……之后们让grunt/gulp来依次执行这些task，而且让整个流程自动化，所以grunt/gulp也被称为前端自动化任务管理工具。

问：什么时候用grunt/gulp？

答：1、工程模块依赖简单。2、只需要进行简单的合并、压缩。

问：grunt/gulp和webpack有什么不同？

答：grunt/gulp更加强调的是前端流程的自动化，模块化不是它的核心。webpack更加强调模块化的开发管理，而文件压缩合并、预处理等功能是他附带的功能。

webpack打包

一般webpack的目录结构有：

  • webpack
    • dist ——用来存放打包好的文件，用于网站发布
      • bundle.js——打包生成的文件
    • src——用于开发
      • mathUtils.js——存放函数，并且使用模块化的方式导出
      • main.js——导入模块化后的方法，并且使用
    • index.html——用script标签引入打包后生成的文件

打包命令：webpack 要打包的文件地址 打包成的目标文件地址

在打包时只需要打包入口文件，他会自动检测其是否导入了哪些模块并且一同进行打包。

webpack配置

在项目根目录下创建webpack.config.js文件（该名称固定）

//webpack.config.js
//导入path依赖，用于动态获取绝对路径
const path = require('path');

module.exports = {
    //要打包的目标文件
    entry:'./src/main.js',
    //要生成的目标文件
    output:{
        //动态获取绝对路径
        path:path.resolve(__dirname,'dist'),
        filename:'bundle.js'
    },
}

想要导入path依赖需要先使用npm init命令来配置package.json

配置完以上两种文件后，就可以直接在项目根目录下直接使用webpack命令来对项目进行打包，省去了后面写路径的步骤

//package.json
{
  "name": "testwebpack",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
    //配置npm指令
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build":"webpack"
  },
  "author": "",
  "license": "ISC"
}

我们可以在package.json中配置指令，当我们使用npm run 指令名时，就可以直接执行该指令名后面所对应的指令，如我们在上方的文件中配置了

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build":"webpack"
  },

当我们想执行webpack指令时，可以用npm run build来代替。

在终端中执行webpack指令会默认调用全局的webpack依赖，而使用在package.json中配置的命令会调用项目中安装的依赖。

问：为什么要在package.json中配置对应的指令来代替webpack指令？

答：在package.json中配置的指令，会优先查找局部的也就是本项目依赖的webpack包，再去查找全局的webpack包，能有效避免由于项目依赖的webpack包的版本与全局拥有的webpack包的版本不同而产生指令不同等一系列问题。

在项目目录下使用npm install webpack@3.6.0 –save-dev指令可以安装局部的webpack包，–save-dev参数表示此依赖为开发时依赖

loader配置使用

loader是webpack中非常核心的一个概念，能帮助我们完成TS、ES6转ES5，scss、less转css，jsx、vue文件转js文件等一些转化。

css

  1. 通过npm安装需要使用的loader。
  2. 在webpack.config.js中的module关键字下进行配置。
npm install --save-dev css-loader
npm install --save-dev style-loader
//webpack.config.js
const path = require('path');

module.exports = {
    entry:'./src/main.js',
    output:{
        path:path.resolve(__dirname,'dist'),
        filename:'bundle.js'
    },
    //配置loader
    module:{
        rules:[
            {
                //匹配文件后缀
                test:/\.css$/,
                //调用loader的顺序
                use:['style-loader','css-loader']
            }
        ]
    }
}

注意：

  1. css-loader只负责css文件的加载
  2. style-loader负责文件样式转成dom样式
  3. 使用多个loader时，使用顺序从右到左，配置顺序不可随意调换！

若装了loader后依旧不生效，可能是loader版本与webpack版本不匹配

less

  1. 编写less文件

    //special.less
@fontSize:50px;
body{
    font-size: @fontSize;
}

  2. 导入less文件

    //main.js
//依赖less文件
require('./css/special.less');

  3. 安装loader

    //less和lessloader的版本根据需要可自选
npm install --save-dev less-loader@4.1.0 less@3.9.0

  4. 配置loader

    const path = require('path');

module.exports = {
    entry:'./src/main.js',
    output:{
        path:path.resolve(__dirname,'dist'),
        filename:'bundle.js'
    },
    //配置loader
    module:{
        rules:[
            {
                //匹配less后缀的文件
                test:/\.less$/,
                //使用对象的方式传入loader配置，可以为他传入其他参数
                //依赖调用顺序从下往上
                use:[
                    {
                        loader:'style-loader'
                    },
                    {
                        loader:'css-loader'
                    },
                    {
                        loader:'less-loader'
                    }
                ]
            }
        ]
    }
}

  5. 使用npm run build指令打包项目

图片

  1. 在src目录下创建img文件夹用于存放文件

  2. 在想要图片生效的地方使用url引入文件

    //normal.css
body {
    background: url('../img/timg.jpeg');
}

  3. 安装相关依赖

    npm install url-loader --save-dev

  4. 配置loader

    //webpack.config.js
const path = require('path');

module.exports = {
    entry:'./src/main.js',
    output:{
        path:path.resolve(__dirname,'dist'),
        filename:'bundle.js'
    },
    //配置loader
    module:{
        rules:[
            {	//匹配文件名后缀
                test: /\.(png|jpg|gif|jpeg)$/i,
                use: [
                //用对象配置依赖
                  {
                    loader: 'url-loader',
                    //限制图片大小为81920k（80kb）
                    options: {
                      limit: 8192*10
                    }
                  }
                ]
              }
        ]
    }
}

    • 当加载图片时大小小于limit的限制时，会将图片编译成base64字符串形式

    • 当加载图片的大小大于limit的限制时，就需要使用file-loader进行加载

      • ```
        npm install –save-dev file-loader
        
- 由于用过file-loader打包后会在我们output指定的路径中生成一张新的图片，我们需要指定此图片所在的路径，可以在output中添加配置属性publicPath，此时就可以指定打包完后的图片的路径，在调用时会自动为图片加上dist/的前缀

  ```js
  const path = require('path');
  
  module.exports = {
      entry:'./src/main.js',
      output:{
          path:path.resolve(__dirname,'dist'),
          filename:'bundle.js',
          publicPath:'dist/'
      },
      //配置loader
      module:{
          rules:[
              {
                  test: /\.(png|jpg|gif|jpeg)$/i,
                  use: [
                    {
                      loader: 'url-loader',
                      options: {
                        limit: 8192*10,
                        //限制file-loader生成的文件名（在img文件夹中+原名+8位hash值+拓展名）
                        name:'img/[name].[hash:8].[ext]'
                      }
                    }
                  ]
                }
          ]
      }
  }

webpack插件

webpack插件（plugin）是对webpack现有功能的各种拓展，如打包优化、文件压缩等。

问：loader和plugin的区别是什么？

答：loader是一个转换器，主要用于转换某些类型的模块；plugin是扩展器，他是对webpack本身的拓展。

使用过程

  1. 通过npm安装需要使用的plugins（某些webpack已经内置的插件无需安装）
  2. 在webpack.config.js中的plugins中配置插件

注释插件

BannerPlugin

此插件是webpack内置插件，能在打包项目后自动为我们添加上注释，来表示版权等信息

使用步骤：

  1. 在webpack.config.js中引入webpack

  2. 在module.exports下配置plugins属性

    …………
const webpack = require('webpack');

module.exports = {
 	………
    plugins:[
        new webpack.BannerPlugin('吴炜瀚记')
    ]
}

主页文件生成

html-webpack-plugin

此插件能让我们在打包时在发布项目的dist文件夹中自动生成一个index.html文件

使用步骤：

  1. 使用npm安装相关插件

    npm install --save-dev html-webpack-plugin

  2. 在webpack.config.js中配置

    1. 引入插件

      const HtmlWebpackPlugin = require('html-webpack-plugin');

    2. 在plugins属性中配置

      plugins:[
    new webpack.BannerPlugin('吴炜瀚记'),
    new HtmlWebpackPlugin()
]

之后打包文件时就会在dist文件夹中自动生成一个index.html文件

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Webpack App</title>
  </head>
  <body>
  <script type="text/javascript" src="bundle.js"></script></body>
</html>

因为我们生成的index.html文件在dist文件夹内，因此我们在这个文件夹内的index.html文件的路径上可以不用再加上**./dist**，可以将output中的publicPath属性去掉

output:{
        path:path.resolve(__dirname,'dist'),
        filename:'bundle.js',
        //publicPath:'dist/'
    },

注意：当我们去掉此配置属性，我们在dist文件夹外的index.html文件中的图片会失效，因为在url中少了这个前缀

但是在此时生成的文件夹中，还缺少一个挂载了Vue实例的div，我们可以通过给调用生成index.html的插件方法传入参数来自动生成这个div

plugins:[
        new webpack.BannerPlugin('吴炜瀚记'),
        new HtmlWebpackPlugin({
            template:"index.html"
        })
    ]

搭建本地服务器

我们每一次进行项目测试时，都需要输入npm run build 来构建项目，这样子会大大影响我们的开发效率

webpack提供了一个可选的基于node.js搭建、内部使用express框架的本地开发服务器，可以实现浏览器自动刷新显示我们修改后的结果

其原理是将我们修改后的代码保存到内存去而不是直接生成到硬盘中，在内存中读取的速度远大于从硬盘中读取

搭建步骤：

  1. npm下载服务器所需文件

    npm install webpack-dev-server --save-dev

  2. 配置webpack.config.js

    ………
module.exports = {
    ………
    devServer:{
        //提供服务的文件夹
        contentBase:'./dist',
        //端口号
        port:8010,
        //页面实时刷新
        inline:true
    }
}

  3. 此时我们可以输入命令webpack-dev-server来运行这个服务器，但此时我们在命令行输入系统会优先查找全局的相关文件，而我们仅安装在了项目中，所以我们还需要在package.json中配置script命令，来使他在查找文件时优先查找项目中的相关文件

    //--open表示服务器启动时自动打开页面
"dev": "webpack-dev-server --open"

  4. 配置完后，我们在开发时就可以使用npm run dev指令来进行开发测试，在我们开发完毕后，才使用npm run build进行打包

配置文件分离

我们在开发时和项目发布时需要的一些配置是不同的，有的配置及其文件在项目发布后是不需要的，有的配置在开发阶段就加上会影响我们对代码的调试，因此我们就需要把他们进行分离。

分离步骤：

  1. 在项目目录下创建build文件夹，里面有三个文件用于存放依赖配置

        rules:[
            {
                test:/\.css$/,
                use:['style-loader','css-loader']
            },
            {
                test:/\.less$/,
                use:[
                    {
                        loader:'style-loader'
                    },
                    {
                        loader:'css-loader'
                    },
                    {
                        loader:'less-loader'
                    }

            },
            {
                test: /\.(png|jpg|gif|jpeg)$/i,
                use: [

                    options: {

                  }

路径别名

我们在移动一个文件时，里面如果有大量的路径引用的话，我们移动后会非常的麻烦，此时我们只需要给引用的目标资源文件所处的文件夹起一个别名，在我们移动时，这个路径会自动锁定到该资源所处的文件夹，就不需要修改资源路径了

设置方法：

  1. 在webpack.base.conf.js中添加resolve的alias中的属性

    ……
resolve: {
    extensions: ['.js', '.vue', '.json'],
    alias: {
      //vuecli2需要将路径写全，vuecli3后就可以在下面的别名中使用上面起完了的别名
      '@': resolve('src'),
      'assets': resolve('src/assets'),
      'components': resolve('src/components'),
    }
  }
……

  2. 引入时可以用起的别名代替路径

    //import MainTabBar from '../components/MainTabBar'
import MainTabBar from 'components/MainTabBar'

<img slot="item-icon-active" src="../assets/img/tabbar/home_active.svg" alt="">
-->

Webpack打包原理

总体分三步：

  1. 建立自执行函数的框架，收集文件中所有的依赖
  2. 将ES6转成ES5然后写入键值表里
  3. 替换require与exports
  4. 自执行打包运行

模拟替换exports

我们可以编写一个自执行函数，将exports作为第一个参数，将传来的字符串代码作为第二个参数，在执行函数中使用eval()执行

<script>
    var exports = {};
    (function (exports,code) {
        eval(code)
    })(exports ,'exports.default = function(a,b){return a+b}')
    console.log(exports.default(1,4))
</script>

模拟替换require

require就是根据提供的file名称加载对应的模块

<script>
  // 模拟require方法
  function require(file) {
    var exports = {};
    (function (exports,code) {
      eval(code)
    })(exports ,'exports.default = function(a,b){return a+b}')
    return exports;
  }
  var add = require("./add.js").default;
  console.log(add(1,3));
</script>

将代码整理成key-value表并且自执行

可以将引入模块的文件名以及内部字符串整理成一张key-value的表，这样通过调用不同的文件名就可以加载不同的模块

{
	"index.js":"var add = require('./add.js').default;console.log(add(1,3))",
	"add.js":"exports.default = function(a,b){return a+b}"
}

接着可以再编写一个自执行函数来调用

(function(list){
  // 模拟require方法
  function require(file) {
    var exports = {};
    (function (exports,code) {
      eval(code);
    })(exports ,list[file]);
    return exports;
  };
  require('index.js')
})({
  "index.js":"var add = require('add.js').default;console.log(add(1,3))",
  "add.js":"exports.default = function(a,b){return a+b}"
});

执行过程：

  1. 自执行第一次到require(‘index.js’)
  2. 在参数中查找index.js，并且执行其中代码作为exports返回执行
  3. 查找到index.js中也有require(‘add.js’)，就找到add.js中的代码赋予exports对象
  4. 返回index.js中执行add(1,3)
