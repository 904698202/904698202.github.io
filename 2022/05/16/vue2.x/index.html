MVVM

View

视图层，在前端开发中也就是DOM层，用于给用户展示各种信息。

Model

数据层，数据可能是我们固定的死数据，更多的是我们服务器从网络上请求下来的数据。

ViewModel

视图模型层，是View和Model沟通的桥梁

一方面实现了Data Binding（数据绑定），将Model的改变实时反映到View中；另一方面实现了DOM Listener（DOM监听），当DOM发生一些事件（点击、滚动、触摸）时，可以随时监听到并且在需要的情况下改变对应的Data。

在下面的计数器实例中，View是我们的DOM，Model是我们抽离出来的obj，ViewModel是我们创建的Vue对象实例。

首先，ViewModel通过Data Binding让obj中的数据实时的在DOM中显示

其次，ViewModel通过DOM Listener来监听DOM事件，并且通过methods中的操作来改变obj中的数据

<!--View-->
<div id="app">
    <h2>当前计数：{{counter}}</h2>
    <button @click="add">+</button>
    <button @click="sub">-</button>
</div>

<script>
    //Model
    const obj = {
        counter:0,
        message:'计数器'
    }
    //ViewModel
    const app = new Vue({
        el:"#app",
        data:obj,
        methods:{
            add:function(){
                this.counter++;
            },
            sub:function(){
                this.counter--;
            }
        }
    })
</script>

Vue生命周期

生命周期：事物从诞生到消亡的整个过程

当new了一个Vue对象时，就创建了一个生命周期，在生命周期中，有一系列预定义的函数，在这些函数执行的过程中，又插入了一些钩子函数，每一个钩子函数被执行我们就可以知道系统执行到哪些函数

get和set方法
<div id="app">
    <h2>{{fullName}}</h2>
</div>
<script>
    const app = new Vue({
        el:'#app',
        data:{
            firstName:'Lebron',
            lastName:'James'
        },
        computed:{
            fullName:{
                //用于主动修改数据，一般为空
                set:function(){},
                //用于返回数据
                get:function(){
                    return this.firstName+ " " +this.lastName;
                }
            }
        }
</script>

get()什么时候执行？

  • 初次读取时会执行一次。
  • 当依赖的数据发生改变时会被再次调用。

如果计算属性中只有get()方法，计算出的属性为只读属性，那么就可以直接简写为

computed:{
    fullName:function(){
        return this.firstName+ " " +this.lastName;
    }
}

如果计算属性有set()方法，那么我们就可以对这个计算出的属性进行修改。

<div id="app">
    <h2>{{fullName}}</h2>
</div>
<script>
    const app = new Vue({
        el:'#app',
        data:{
            firstName:'Lebron',
            lastName:'James'
        },
        computed:{
            fullName:{
                //传入一个新的参数，对计算属性进行修改
                set:function(newValue){
                    console.log(newValue);
                    const name = newValue.split(' ');
                    this.firstName=name[0],
                    this.lastName=name[1]
                },
                //用于返回数据
                get:function(){
                    return this.firstName+ " " +this.lastName;
                }
            }
        }
</script>

计算属性与Mustache语法的对比

相同点：都可以将两段数据拼接起来

不同点：Mustache语法每次使用函数的方法拼接都要调用一次函数，计算属性拼接只需要调用一次函数

因为存在缓存的原因，当计算的数据没有发生更改时，计算属性是不会再次进行计算，因此计算属性的性能更高

computed实现与methods实现相比

相同点：都能实现属性的计算，使用返回值返回计算后的属性。

不同点：与methods实现相比，每次页面刷新，我们定义的方法都要重新调用一次，而计算属性内部有缓存机制，能复用得到的结果，效率更高，调试更方便。

注意点：

  • 计算属性最终会出现在app上，直接读取使用即可。
  • 如果计算属性要被修改，那必须写set函数去响应修改，且set中要引起计算时依赖的数据发生改变。

watch属性

监视属性watch：

1.当被监视的属性变化时, 回调函数自动调用, 进行相关操作

2.监视的属性必须存在，才能进行监视！！

3.监视的两种写法：

(1).new Vue时传入watch配置

(2).通过app.$watch监视

  <div id="root">
    <h2>今天天气很{{info}}</h2>
    <button @click="changeWeather">切换天气</button>
  </div>
</body>

<script type="text/javascript">
  Vue.config.productionTip = false //阻止 vue 在启动时生成生产提示。

  const app = new Vue({
    el: '#root',
    data: {
      isHot: true,
    },
    computed: {
      info() {
        return this.isHot ? '炎热' : '凉爽'
      }
    },
    methods: {
      changeWeather() {
        this.isHot = !this.isHot
      }
    },
    /* watch:{
      isHot:{
        immediate:true, //初始化时让handler调用一下
        //handler什么时候调用？当isHot发生改变时。
        handler(newValue,oldValue){
          console.log('isHot被修改了',newValue,oldValue)
        }
      }
    } */
  })

  app.$watch('isHot', {
    immediate: true, //初始化时让handler调用一下
    //handler什么时候调用？当isHot发生改变时。
    handler(newValue, oldValue) {
      console.log('isHot被修改了', newValue, oldValue)
    }
  })
</script>

watch只能监视到第一层的数据，如基本数据类型以及对象或数组的整个是否有改变，若要监视到数组或对象内部的数据变化可以开启深度监视

app.$watch('isHot',{
    immediate:true, //初始化时让handler调用一下
    deep:true,//深度监视
    handler(newValue,oldValue){
        console.log('isHot被修改了',newValue,oldValue)
    }
})

如果我们不需要配置初始化执行与深度监视，可以将监视属性简写成一个函数

watch:{
    //正常写法
    /* isHot:{
		// immediate:true, //初始化时让handler调用一下
		// deep:true,//深度监视
		handler(newValue,oldValue){
		console.log('isHot被修改了',newValue,oldValue)
		}
	}, */
    //简写
    isHot(newValue,oldValue){
        console.log('isHot被修改了',newValue,oldValue,this)
    } 
}

computed和watch的区别

  1. computed能完成的功能，watch都可以完成。
  2. watch能完成的功能，computed不一定能完成。如：watch可以进行异步操作。

注意事项

  1. 所被Vue管理的函数（computed、methods等配置项里面的函数），最好写成普通函数，这样this的指向才是app或组件实例对象。
  2. 所有不被Vue所管理的函数（定时器的回调函数、ajax的回调函数等、Promise的回调函数），最好写成箭头函数，这样this的指向才是app或组件实例对象。

自定义指令

我们自定义一些指令来实现自己想要实现的效果。

我们可以利用vue实例中的directives配置项进行局部指令的绑定或者使用API——**Vue.directive()**进行全局自定义指令的绑定。

自定义指令时可以用回调的方式写也可以用配置对象的方式写，如果要实现一些细节操作的话需要以配置对象的方式去定义指令。

配置对象中常用的3个回调：

  1. bind：指令与元素成功绑定时调用。
  2. inserted：指令所在元素被插入页面时调用。
  3. update：指令所在模板结构被重新解析时调用。

需求1：定义一个v-big指令，和v-text功能类似，但会把绑定的数值放大10倍。

需求2：定义一个v-fbind指令，和v-bind功能类似，但可以让其所绑定的input元素默认获取焦点。

<div id="root">
    <h2>{{name}}</h2>
    <h2>当前的n值是：<span v-text="n"></span> </h2>

    <h2>放大10倍后的n值是：<span v-big="n"></span> </h2>
    <button @click="n++">点我n+1</button>
    <hr/>
    <input type="text" v-fbind:value="n">
</div>
</body>

<script type="text/javascript">
    Vue.config.productionTip = false

    new Vue({
        el:'#root',
        data:{
            name:'尚硅谷',
            n:1
        },
        directives:{
            //big函数何时会被调用？1.指令与元素成功绑定时（一上来）。2.指令所在的模板被重新解析时。
            big(element,binding){
                console.log('big',this) //注意此处的this是window
                element.innerText = binding.value * 10
            },
            fbind:{
                //指令与元素成功绑定时（一上来）
                bind(element,binding){
                    element.value = binding.value
                },
                //指令所在元素被插入页面时
                inserted(element,binding){
                    element.focus()
                },
                //指令所在的模板被重新解析时
                update(element,binding){
                    element.value = binding.value
                }
            }
        }
    })

</script>

组件化

如果将一个页面的所有处理逻辑全部放在一起，那么处理起来就会十分复杂并且不利于后续的管理以及拓展。此时，我们就应该将一个页面拆分成一个个小的功能块，每个功能块完成属于自己的这部分独立功能，有利于项目的管理和维护

Vue的组件化思想

提供了一种抽象，让我们可以开发出一个个独立可复用的小组件来构造我们的应用，任何的应用都会被抽象成一颗组件树

  1. 创建组件构造器
  2. 注册组件

注意：

  1. 当模板内有多个同级的HTML标签时，模板内的HTML代码必须要用一个根元素（如div）包起来，否则模板里面的HTML代码只会生效第一个HTML标签的内容

自定义事件

$emit()

使用$emit()调用父组件在子组件标签上绑定的自定义事件的方式也可以实现子组件给父组件传数据。

注意：如果在父组件上给子组件的标签绑定原生DOM事件，VUE也会把他当成是自定义事件，需要用$emit()触发。如果要避免这种情况应该用.native修饰符修饰v-on。

子组件想要给父组件传递数据分为三步：

  1. 在子组件中利用$emit()给父组件发射一个自定义事件
  2. 在父组件的子组件标签上用v-on绑定上$emit()发射来的自定义事件
  3. 在父组件中定义methods来处理从子组件传来的自定义事件

理解二：

  1. 在父组件中为子组件的标签绑定一个自定义事件与这个自定义事件被触发后的处理函数
  2. 在子组件中使用$emit()去触发绑定在自己标签上的自定义事件

消息订阅与发布

使用消息订阅与发布也可以实现任意组件间通讯，想要使用这种方法需要借助pubsub-js库。

npm i pubsub-js

安装完这个库后，我们在需要使用到库里面API的组件中引入这个库即可。

import pubsub from 'pubsub-js'

引入完成后的pubsub是一个对象，这个对象上有三个API可以使用：

  1. subscribe——订阅频道
  2. publish——发布消息
  3. unsubscribe——取消订阅

subscribe

该方法有两个参数：

  1. 订阅频道的名字
  2. 该频道收到消息后执行的回调函数，这个回调也会收到两个参数：
    1. 订阅频道的名字
    2. 收到的数据

并且该方法会有一个返回值，作为频道订阅完成后的ID（好比定时器的定时器变量），可用于后续取消订阅。

publish

该方法有两个参数：

  1. 消息要发往的频道名字
  2. 消息数据

unsubscribe

该方法的参数就是上方使用subscribe订阅频道后生成的订阅ID。

主要用在不需要使用这个频道后的解除订阅的操作。

				console.log(this)

plugin插件

Vue中的插件是一个对象，在这个对象中使用install()方法进行插件的定义，可以为Vue添加全局的功能。

插件的功能没有严格的限制：

  1. 可以用来添加全局方法或property——vue-custom-element
  2. 添加全局资源：指令/过滤器/过渡等。如 vue-touch
  3. 通过全局混入来添加一些组件选项。如 vue-router
  4. 添加 Vue 实例方法，通过把它们添加到 Vue.prototype 上实现。
  5. 一个库，提供自己的 API，同时提供上面提到的一个或多个功能。如 vue-router
//plugins.js
//定义插件
export default {
	install(Vue,x,y,z){
		console.log(x,y,z)
		//全局过滤器
		Vue.filter('mySlice',function(value){
			return value.slice(0,4)
		})

		//定义全局指令
		Vue.directive('fbind',{
			//指令与元素成功绑定时（一上来）
			bind(element,binding){
				element.value = binding.value
			},
			//指令所在元素被插入页面时
			inserted(element,binding){
				element.focus()
			},
			//指令所在的模板被重新解析时
			update(element,binding){
				element.value = binding.value
			}
		})

		//定义混入
		Vue.mixin({
			data() {
				return {
					x:100,
					y:200
				}
			},
		})

		//给Vue原型上添加一个方法（app和vc就都能用了）
		Vue.prototype.hello = ()=>{alert('你好啊')}
	}
}

$nextTick

如果我们在方法中对页面相关的数据进行修改，此时Vue并不会在每次修改数据后就重新渲染数据，而是等方法全部都执行完了再进行页面的渲染。

此时如果我们在方法中的某些语句依赖于修改后的页面执行就会执行无效，因为此时修改后的页面还没有被渲染。

为了应对这种情况，vue给我们提供了一个API——$nextTick()

这个API的参数是一个回调函数，并且此回调函数会在下一次DOM节点更新后再执行。

    handleEdit(item) {
        //我们通过方法修改了页面的修改输入框的状态
        //但是页面不会马上将文本框渲染成输入框，而是等待此方法内所有代码执行完再进行渲染
      if (item.hasOwnProperty('isEdit')) {
        item.isEdit = true
      } else {
        this.$set(item,'isEdit',true)
      }
      //我们想对渲染完成后的页面进行操作时会失败，因为此时方法内代码还未执行完成，输入框还未渲染再页面上
      //this.$refs.editInput.focus()
        
      //解决方法一：套上一个定时器，放入宏队列中执行
      /**
      setTimeout(() => {
        this.$refs.editInput.focus()
      },0)*/
        
      //解决方法二：$nextTick()
      this.$nextTick(()=>{
        this.$refs.editInput.focus()
      })
    },
  },
};
</script>

组件中的CSS

scope

在脚手架打包后，所有的组件中的CSS样式都会汇总在一起，此时难免有类名相同等问题出现，此时样式生效顺序按照后引入的生效。

为了避免这种问题，可以在组件的style标签上加上scope属性，此属性可以时改样式仅仅在该组件中生效。

其原理是在这个样式的标签上加了一个唯一的标签用来识别样式。

lang

在组件的style标签上可以加上lang属性，并且为其赋值为less或sass，配合webpack的loader可以让Vue支持less或sass语法。

动画与过渡

在VUE中，可以使用transition组件标签将要添加动画效果的元素包裹起来。

该组件能给任意元素和组件添加进入和离开的过渡动画：

  • 条件渲染（v-if）
  • 条件展示（v-show）
  • 动态组件
  • 组件根节点

动画效果

我们可以为这个标签的内置类名添加动画，期间涉及两个类名：v-enter-active（进入时触发），v-leave-active（离开时触发）

类名 作用
v-enter-active 定义元素进入过程中触发的样式
v-leave-active 定义元素离开过程中触发的样式 
<style>
  .msg-box {
    width: 100%;
    background: red;
  }
    /*用过渡组件的内置类名定义过渡动画*/
    /*如果transition有一个name属性，那么该类名的v将会被替换成name的属性值*/
    /*如name="msg"，则该内置类名为msg-enter-active*/
    /*使用name的好处在于存在多个transition组件标签时，能决定此组件到底对应哪个动画，也有利于代码复用*/
  .v-enter-active{
    animation: donghua 0.5s linear;
  }
  .v-leave-active{
    animation: donghua 0.5s reverse linear;
  }

  @keyframes donghua {
    from {
      transform: translateX(-100%);
    }
    to {
      transform: translateX(0);
    }
  }
</style>

    <!--为组件标签写一个appear属性能决定此动画在组件渲染完成时会不会先执行一次-->

过渡效果

在上面的基础上，我们可以不写动画，改成过渡的方式实现同样的效果，其中除了上面涉及到的两个类名，还额外涉及到4个类名：

类名 作用
v-enter 定义元素进入的起点
v-enter-to 定义元素进入的终点
v-leave 定义元素离开的起点
v-leave-to 定义元素离开的终点 
.msg-box {
  width: 100%;
  background: red;
  /* 添加过渡：可以挑出来单独写 */
  /* transition: 0.5s linear; */
}
.v-enter-active,.v-leave-active {
  transition: 0.5s linear;
}
.v-enter {
  transform: translateX(-100%);
}
.v-enter-to {
  transform: translateX(0);
}
.v-leave {
  transform: translateX(0);
}
.v-leave-to {
  transform: translateX(-100%);
}

当插入或删除包含在 transition 组件中的元素时，Vue 将会做以下处理：

  1. 自动嗅探目标元素是否应用了 CSS 过渡或动画，如果是，在恰当的时机添加/删除 CSS 类名。
  2. 如果过渡组件提供了 JavaScript 钩子函数，这些钩子函数将在恰当的时机被调用。
  3. 如果没有找到 JavaScript 钩子并且也没有检测到 CSS 过渡/动画，DOM 操作 (插入/删除) 在下一帧中立即执行。(注意：此指浏览器逐帧动画机制，和 Vue 的 nextTick 概念不同)

注意：

  1. 每一个transition标签内只能有一个元素，如果要实现多元素动画需要使用transition-group，并且给每个元素标签上加上一个key值（必须）。

    <template id="cpn">
  <div>
    <button @click="changeShowState">切换状态</button>
    <transition-group appear>
    <!--两个元素互斥出现-->
    <div v-show='isShow' key="1" class="msg-box">
      {{msg}}
    </div>
    <div v-show='!isShow' key="2" class="msg-box">
      {{msg}}
    </div>
  </transition-group>
  </div>
</template>

集成第三方库

我们可以不写过渡的CSS样式，而借助第三方库如：animate.css来完成动画效果。

我们只需要在自定义过渡的类名中写入第三方库动画的类名。

注意：

  1. 这边动画不生效可能跟库的版本有关，可能早点版本的库内没有现在的一些标签，所以引入最新的库就行。
  2. 这些动画的类名一般要有一个animated，第二个类名则是动画的名字。
<transition-group 
  appear 
  enter-active-class="animate__animated  animate__backInDown"
  leave-active-class="animate__animated  animate__backOutDown"
>
<div v-show='isShow' key="1" class="msg-box">
  {{msg1}}
</div>
<div v-show='!isShow' key="2" class="msg-box">
  {{msg2}}
</div>
</transition-group>
脚手架安装

单个元素动画

    <div class="count-item">1</div>
</transition>
多个元素动画
<transition-group appear enter-active-class="animate__animated animate__backInDown">
    <div class="count-item" key="1">2</div>
    <div class="count-item" key="2">2</div>
    <div class="count-item" key="3">2</div>
    <div class="count-item" key="4">2</div>
</transition-group>
菜单页面过度
  • 可以配合transition进行整个页面切换的过渡
  • 当页面切换时需要将mode设置为out-in，否则会有两个页面同时存在的情况发生
  • 当该页面有使用keep-alive进行缓存时，需要包括在该标签外，否则会导致keep-alive失效，无法触发actived等生命周期钩子
<transition mode="out-in"
            enter-active-class="animate__animated  animate__fadeInUp"
            leave-active-class="animate__animated  animate__fadeOutDown">
    <keep-alive>
        <router-view exclude="编辑公告" style="height: 500px"></router-view>
    </keep-alive>
</transition>

Vue程序运行过程

  1. 保存为template模板文件
  2. 将template模板解析成AST抽象语法树
  3. 将AST编译成render函数
  4. 通过此render函数将这些template模板编译成一个虚拟dom树
  5. 将这些虚拟dom树来转换成真实dom渲染到页面上

问：runtime-compiler和runtime-only的区别是什么？

答：

  • 使用runtime-compiler的Vue程序运行过程为：template—AST—render—VDom—UI
  • 使用runtime-only的的Vue程序运行过程为：render—VDom—UI
  • runtime-only由于省略了前两个运行步骤，因此他的性能更高，代码量也更少

因此我们在使用脚手架开发时通常会选择runtime-only来开发。

render函数

也被称为渲染函数，负责将template模板编译成一个虚拟DOM树

此函数中有一个createElement属性，可以用来生成一个DOM

createElement属性

createElement属性中可以传入三个参数：

  1. 要生成的HTML标签
  2. 传入一个对象作为此HTML标签的属性
  3. 传入一个数组作为此HTML标签的内容（可以再次传入一个createElement函数来进行嵌套）
new Vue({
  el: '#app',
  // components: { App },
  // template: '<App/>'
  render: function(createElement){
    return createElement('h2',
            {class:'box'},
            ['Hello world',
          createElement('button',['按钮'])])
  }
})

我们也可以直接为这个属性传入一个组件对象

const cpn = {
  template:`
    <div>
      <h2>{{message}}</h2>
      <button>按钮</button>
    </div>
  `,
  data(){
    return {
      message:"hello World"
    }
  }
}

/* eslint-disable no-new */
new Vue({
  el: '#app',
  render:function(createElement){
    return createElement(cpn);
  }
  //ES6写法
  //render:createElement=>createElement(cpn)
})

我们如果将引入的.vue文件作为一个参数传入createElement中，他已经被我们webpack的vue-template-compiler编译成render函数，帮我们省去了Vue程序运行过程的前两步，这也是我们开发选择runtime-only开发的原因之一

import Vue from 'vue'
import App from './App'

Vue.config.productionTip = false

/* eslint-disable no-new */
new Vue({
  el: '#app',
  render:createElement=>createElement(App)
})
