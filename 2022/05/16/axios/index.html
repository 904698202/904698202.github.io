Axios

前言

json-server

我们可以用json-server搭建一个虚拟的API以供网络测速。

npm i -g json-server

新建一个json-server文件夹，在里面创建db.json用于存放虚拟的json数据

之后我们就可以根据命令行输出的几个API来取得这几个json数据。

PS D:\workplace\web\json-server> json-server --watch db.json

支持使用rest API的请求方式，如：

功能特点

  • 在浏览器中发送XMLHttpRequest请求
  • 在node.js中发送http请求
  • 支持Promise API
  • 拦截请求和响应
  • 转换请求和响应数据

使用方式

  1. 安装axios：npm install axios –save

  2. 引用插件，并编写一个请求

    import Vue from 'vue'
import App from './App'
import router from './router'
//1、引用
import axios from 'axios'

Vue.config.productionTip = false

/* eslint-disable no-new */
new Vue({
  el: '#app',
  router,
  render: h => h(App)
})
//2、发送请求
axios({
  url:'http://123.207.32.32:8000/home/multidata'
}).then(res => {
  console.log(res)
})

axios()方法默认返回一个Promise对象，因此我们可以直接用then()方法去对数据进行处理

参数类型

axios()方法可以传入一个对象，对象内有不同的参数，常用的数据类型有：

参数 作用
url（必须） 指定发送请求的目标接口
method（可选） 指定发送请求的类型，如果不填默认为GET请求
baseURL（可选） 指定一个基础url，之后在请求url时就可以省去这一段指定好的url，也可以把这个参数拉出方法外单独定义
params（可选） 当我们使用get请求时，需要在url后面拼接参数，可以直接写在url参数内，也可以写在params参数中
transformRequest:[function(data){}]（可选） 请求前数据处理，在回调函数中处理完的数据需要return才能生效
transformResponse:[function(data){}]（可选） 请求后数据处理，在回调函数中处理完的数据需要return才能生效
headers:{‘x-Requested-With’:’XMLHttpRequest’}（可选） 自定义请求头信息（token会放到这里面）
paramsSerializer:function(params){} 查询对象序列化函数
data（可选） 请求体，对应post请求
timeout（可选） 超时设置
withCredentials（可选） 跨域是否携带Token
adapter:function(resolv,reject,config){}（可选） 自定义请求处理
auth（可选） 身份验证信息
responseType（可选） 响应数据格式 
//指定基础URL
axios.defaults.baseURL='http://123.207.32.32:8000';
//默认方式
axios({
  //baseURL:'http://123.207.32.32:8000',
  url:'/home/multidata',
  method:'get',
    
}).then(res => {
  console.log(res)
})
//axios.get()方法
axios.get('/home/data?type=sell$page=3').then(res => {
  console.log(res)
})
axios({
  url:'/home/data',
  method:'get',
  //get方法携带的参数
  params:{
    type:'sell',
    page:3
  }
}).then(res => {
  console.log(res)
})

全局配置

开发中有很多参数都是固定的，需要在很多地方重复使用，我们可以把他们抽取到axios()外面单独定义，在axios()内就可以省略这些配置

全局配置的优先级低于使用create创建的实例对象中的配置。

//指定基础URL
axios.defaults.baseURL='http://123.207.32.32:8000';
//指定超时时间（毫秒，如果请求时间超过则报错）
axios.defaults.timeout=5000

请求别名

为方便起见，为所有支持的请求方法提供了别名

axios.request(config)

axios.get(url[, config])

axios.delete(url[, config])

axios.head(url[, config])

axios.options(url[, config])

axios.post(url[, data[, config]])

axios.put(url[, data[, config]])

axios.patch(url[, data[, config]])

注意：在使用别名方法时，url、method、data 这些属性都不必在配置中指定。

并发请求

当我们在开发中，需要对两个接口进行访问，并且都访问到结果才继续下一步的处理

与Promise.all([])类似，我们可以使用axios.all([])来处理并发请求

axios.all([
  axios({
    url:'/home/multidata',
    method:'get'
  }),
  axios({
    url:'/home/data',
    method:'get',
    params:{
      type:'sell',
      page:3
    }
  })
]).then(res =>{
  console.log(res[0],res[1])
})

不同在于，axios返回的结果可以用spread()方法展开

axios.all([
  axios({
    url:'/home/multidata',
    method:'get'
  }),
  axios({
    url:'/home/data',
    method:'get',
    params:{
      type:'sell',
      page:3
    }
  })
]).then(axios.spread((res1,res2) => {
  console.log(res1);
  console.log(res2);
}))

axios实例

我们在平时使用axios()发送请求的时候其实是用的一个全局的axios实例，其中defaults的配置也是全局的

但当我们有向多个不同的域名发送请求的需求的时候，通过全局配置的axios方法就会导致请求冲突，此时我们可以创建单个实例来处理这种分布式请求

//使用create()方法创建一个axios实例
const instance1 = axios.create({
  baseURL:'http://123.207.32.32:8000',
  timeout:5000
})

instance1({
  url:'/home/multidata'
}).then(res => {
  console.log(res);
})

instance1({
  url:'/home/data',
  method:'get',
  params:{
    type:'sell',
    page:3
  }
}).then(res => {
  console.log(res)
})

一般情况下我们都是使用这种方式来封装axios网络请求的实例

模块封装

一般我们要在组件中封装axios网络请求都是在Vue的生命周期函数created中

<template>
  <div id="app">
    <p>{{res}}</p>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  name: 'App',
  data(){
    return {
      res:''
    }
  },
  created(){
    axios({
      baseURL:'http://123.207.32.32:8000',
      url:'/home/multidata'
    }).then(res => {
      this.res = res
    })
  }
}
</script>

多个参数

但是很少直接在组件中引入axios这种第三方框架，为了避免更换技术栈的风险，我们一般在src下新建一个network文件夹用于存放对于网络请求的封装

//src/network/request.js
//1、引入axios框架
import axios from 'axios'
//2、封装一个resquest请求，接收配置，以及成功与失败的回调函数
export function request(config,success,failure){
  //3、使用create()创建一个axios实例，里面写上一些基础配置
  const instance = axios.create({
    baseURL:'http://123.207.32.32:8000',
    timeout:5000
  })
  //4、调用创建的axios实例来接收组件传来的配置，捕获成功与失败的回调函数，发送真正的网络请求
  instance(config)
  .then(res => {
    //5.1、执行组件处理成功响应数据的回调函数
    success(res)
  })
  .catch(err => {
    //5.2、执行组件处理失败响应数据的回调函数
    failure(err)
  })
}
//App.vue
<script>
//引入封装好的网络请求模块
import { request  } from "./network/request";

export default {
  name: 'App',
  data(){
    return {
      res:''
    }
  },
  created(){
    request({
      url:'/home/multidata'
    }, res => {
      this.res = res
    })
  }
}
</script>

单个参数

我们可以只让配置好的网络请求模块只接收一个config参数，在这个参数中分为baseConfig和回调函数

import axios from 'axios'

export function request(config){
  const instance = axios.create({
    baseURL:'http://123.207.32.32:8000',
    timeout:5000
  })

  instance(config.baseConfig)
  .then(res => config.success(res) )
  .catch(err => config.failure(err) )
}
<script>
import { request  } from "./network/request";

export default {
  name: 'App',
  data(){
    return {
      res:''
    }
  },
  created(){
    request({
      baseConfig:{
        url:'/home/multidata'
      },
      success(res){
        console.log(res);
      },
      failure(err){
        console.log(err)
      }
    })
  }
}
</script>

Promise写法

import axios from 'axios'

export function request(config){
    //返回一个Promise，通过Promise的resolve和reject来传递回调数据
  return new Promise((resolve,reject) => { 
    const instance = axios.create({
      baseURL:'http://123.207.32.32:8000',
      timeout:5000
    })
	//网络请求执行Promise自带的回调函数
    instance(config)
    .then(res => resolve(res) )
    .catch(err => reject(err) )
  })
}
<script>
import { request  } from "./network/request";

export default {
  name: 'App',
  data(){
    return {
      res:''
    }
  },
  created(){
      //直接使用then与catch来处理数据
    request({
        url:'/home/multidata'
    }).then(res => {
      this.res = res
    }).catch(err => {
      console.log(err)
    })
  }
}
</script>

axios框架最终封装方案

axios框架在源码中，返回的本来就是一个Promise函数，因此我们可以在上面的Promise写法上面直接删去对Promise的封装，直接执行玩网络请求，将这个请求的结果返回，在组件中就直接可以用Promise的方式去调用

import axios from 'axios'

export function request(config){
    const instance = axios.create({
      baseURL:'http://123.207.32.32:8000',
      timeout:5000
    })

    return instance(config)
}

这样，若我们日后更换了网络请求框架的时候，如果返回的结果不是一个Promise，就可以用上面Promise的写法去自己封装一个

拦截器

在我们请求成功、请求失败、响应成功、响应失败这四种情况下，可以创建Axios拦截器对数据进行拦截、操作

拦截器方法 作用
axios.interceptors.request.use() 拦截全局实例下的请求成功与请求失败的数据
axios.interceptors.response.use() 拦截全局实例下的响应成功与响应失败的数据

request拦截器

import axios from 'axios'

export function request(config){
    const instance = axios.create({
      baseURL:'http://123.207.32.32:8000',
      timeout:5000
    })
	
    //对请求配置进行拦截
    instance.interceptors.request.use(config => {
      console.log(config);
      //操作完需要把配置数据返回出去
      return config
    },err => {
      console.log(err);
    })

    return instance(config)
}

使用场景

  1. 当config中一些信息不是我们想要的，我们就可以在拦截器里面进行配置、转化
  2. 每次在网络请求时，希望在页面中显示一个loading的图标
  3. 某些网络请求（例如登录）必须携带一些必须的特殊信息，如果没有这些信息，可以将其拦截，然后跳转到一个特定的地方（比如登录页面）

response拦截器

import axios from 'axios'

export function request(config){
    const instance = axios.create({
      baseURL:'http://123.207.32.32:8000',
      timeout:5000
    })

    instance.interceptors.response.use(result => {
      console.log(result);
      //对响应数据操作完后也要把数据返回出去，否则前端的then中取得的响应结果是undefined
      return result.data
    },err => {
      console.log(err);
    })

    return instance(config)
}

注意：

  1. 若有多个拦截器存在，请求拦截器逆序执行，响应拦截器顺序执行。
  2. 如果使用create创建了的实例里面定义了拦截器，那么在外部定义的全局拦截器将被忽略。
  3. 拦截器可以赋值给一个标记变量，然后用拦截器中的eject()方法去执行这个方法，就可以取消拦截器。（如同定时器）
  4. 拦截器基于Promise，响应拦截器将Promise的状态改变为Resolved，响应响应拦截器执行成功或失败的回调将由response的结果决定，一但有一个拦截器将promise的状态改变为Rejected，他再执行响应拦截器会执行错误位置的回调。

问：拦截器与transformRequest/transformResponse的差别？

答：

相同点：都能在发送请求前/接收到响应后为数据进行拦截处理。

不同点：

  • 拦截器支持所有网络请求方法，而transformRequest/transformResponse只能支持post、put、patch方法。
  • 拦截器能对网络请求的所有配置进行修改，而transformRequest/transformResponse只能对data与headers进行操作。
